### 1. Archivos TEAL

- **Qué son:** TEAL (Transaction Execution Approval Language) es el lenguaje de scripts utilizado en **Algorand** para escribir contratos inteligentes. Los archivos TEAL contienen el código que define las condiciones bajo las cuales una transacción será aprobada o rechazada.
- **En blockchain:** Se usan para crear **Algorand Smart Contracts (ASC1)**, que son seguros y deterministas.
- **Ejemplo práctico:** En tu proyecto educativo, podrías usar TEAL para definir reglas como: _“El token se entrega solo si el estudiante completa el curso y pasa la evaluación”_.
### 2. Lenguaje ensamblador

- **Qué es:** Es un lenguaje de bajo nivel que se aproxima al código máquina. En blockchain, algunos lenguajes para contratos inteligentes (como TEAL o EVM bytecode) tienen una sintaxis similar a ensamblador.
- **En blockchain:** Permite optimizar contratos inteligentes para que sean más eficientes y consuman menos recursos.
- **Ejemplo práctico:** Si necesitas un contrato muy ligero para validar identidad en tu plataforma educativa, podrías usar instrucciones tipo ensamblador para reducir costos.
### 3. Contratos

- **Qué son:** Son **contratos inteligentes (smart contracts)**, programas que se ejecutan en la blockchain y cumplen condiciones predefinidas sin intermediarios.
- **En blockchain:** Garantizan transparencia, seguridad y automatización.
- **Ejemplo práctico:** Un contrato que libere tokens educativos cuando el estudiante suba un trabajo original (control de plagio).
### 4. Bajo costo

- **Qué significa:** En blockchain, se refiere a redes o soluciones que permiten transacciones y ejecución de contratos con **bajas tarifas (gas fees)**.
- **En blockchain:** Algorand, Polygon y Solana son ejemplos de redes con costos bajos.
- **Ejemplo práctico:** Ideal para tu proyecto, porque los estudiantes no tendrían que pagar tarifas altas para interactuar con la plataforma.
### 5. Sandbox

- **Qué es:** Un entorno aislado para probar contratos inteligentes y aplicaciones blockchain sin afectar la red principal.
- **En blockchain:** Permite simular transacciones, probar lógica y detectar errores antes de desplegar en producción.
- **Ejemplo práctico:** Puedes usar el **Algorand Sandbox** para probar tu sistema de recompensas sin gastar tokens reales.
### 6. Testnet

- **Qué es:** Una red de prueba que replica la blockchain principal, pero usa tokens ficticios.
- **En blockchain:** Sirve para experimentar sin riesgos financieros.
- **Ejemplo práctico:** Antes de lanzar tu plataforma educativa en mainnet, podrías usar la testnet para validar la distribución de tokens y contratos.
### 7. Bytecode

- **Qué es:** Código intermedio que se genera al compilar un contrato inteligente antes de ejecutarlo en la máquina virtual de la blockchain.
- **En blockchain:** Es lo que realmente se ejecuta en la red (por ejemplo, EVM para Ethereum).
- **Ejemplo práctico:** Cuando compiles tu contrato en Solidity, se convertirá en bytecode para que la blockchain lo entienda.
