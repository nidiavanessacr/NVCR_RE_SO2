**Sistemas de archivos:** Estructura que organiza y administra cómo se almacenan, acceden y encuentran los datos en un dispositivo de almacenamiento, como un disco duro, SSD o memoria USB. Permite que el sistema operativo interprete los bits de información en archivos significativos y los recupere de manera eficiente, asignando y controlando el espacio en la unidad. Existen diversos sistemas de archivos, cada uno con características específicas y optimizado para ciertos sistemas operativos o tipos de dispositivos.

## **WINDOWS**

**FAT16:** (File Allocation Table 16) es un sistema de archivos antiguo que utiliza una tabla de 16 bits para gestionar el almacenamiento en disco, con un tamaño máximo de partición de 2 GB y soporte para nombres de archivo cortos. ==**(Capacidad: 2GB)**==

**FAT32:** Sistema de archivos muy compatible y antiguo, ideal para dispositivos de almacenamiento extraíbles como memorias USB y tarjetas de memoria, debido a su amplia compatibilidad con diferentes sistemas operativos y dispositivos. ==**(Capacidad: 4GB)**==

**exFAT:** (Extended File Allocation Table) es un sistema de archivos creado por Microsoft para dispositivos de memoria flash, como tarjetas SD y unidades USB, que permite el uso de archivos muy grandes de hasta 16 exabytes, eliminando la limitación de 4 GB de FAT32. ==**(Capacidad: 16EB)**==

**NTFS:** Estructura que organiza y administra cómo se almacenan, acceden y encuentran los datos en un dispositivo de almacenamiento, como un disco duro, SSD o memoria USB. Permite que el sistema operativo interprete los bits de información en archivos significativos y los recupere de manera eficiente, asignando y controlando el espacio en la unidad. Existen diversos sistemas de archivos, cada uno con características específicas y optimizado para ciertos sistemas operativos o tipos de dispositivos. ==**(Capacidad: 16TB)**==

## **LINUX**

**ext4:** Es el cuarto sistema de archivos extendido, un sistema de archivos transaccional (con diario) para Linux que mejora la escalabilidad, confiabilidad y el rendimiento de su predecesor, ext3. Ofrece soporte para volúmenes de hasta 1 exbibyte (EiB) y tamaños de archivo de hasta 16 tebibytes (TiB). ==**(Capacidad: 16TB)**==

**ext3:** (tercer sistema de archivos extendido) es un sistema de archivos de Linux que se caracteriza por el registro por diario (journaling), una función que aumenta la fiabilidad al proteger los datos frente a fallas del sistema o cortes de energía ==**(Capacidad: 2TB)**==
